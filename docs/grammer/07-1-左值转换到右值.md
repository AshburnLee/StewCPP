编译器会做 左值到右值的转换

标准里属于把 glvalue（广义 lvalue） 转成 prvalue（pure rvalue）的一类情况. 具体指：

从一个“指代内存里某块对象”的表达式（左值），读出它里面存的值，得到一个“纯数值、不再绑定那块内存”的临时结果（右值 / prvalue）。

就是 按类型从对象里读一坨比特，当成一个值来用。

例子：

~~~cpp
int a = 10;
int b = a;  // a 是左值；要初始化 b，需要 a 的“值”, 发生左值=>右值转换
int c = tokens[i]; // tokens[i] 是 int& 左值；要拷贝给 c => 读出那一格的 int => 右值
std::cout << tokens[i]; // << 要的是要打印的“值”=> 同样先读出 int
~~~

这里 b = a 并不是“把 a 这个变量整体搬过去”，而是：读 a 里存的 int 值，再拿去初始化 b。

所以：

左值到右值转换 = 从“有地址、可指代的对象”里把当前存的内容读出来，当成按值使用的那个类型的值。


===

表达式
├── glvalue（广义左值，有“身份”、可指代对象或函数）
│   ├── lvalue（左值，如变量名、`*p`、`a[i]` 等可修改或可取址的典型情况）
│   └── xvalue（将亡值，如 `std::move(x)`、返回 `T&&` 的函数调用等， 表示“资源往往马上要移走”、绑定在右值引用上的那种表达式，仍是右值）
│
└── prvalue（纯右值：无身份，多表示“值”本身，字面量、临时对象、多数返回非引用类型的子表达式（如 `a + b`）

另：**rvalue（右值）** = xvalue + prvalue（将亡值 + 纯右值），与“是否为 glvalue”是另一套并集关系。
