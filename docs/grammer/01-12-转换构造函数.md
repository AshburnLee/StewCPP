
当构造函数是*单参数*且不是 explicit 时，它会被当作“转换构造函数” （语义不明确，工程上使用 explicit 以避免）。

explicit 避免*看起来能跑但语义不清*的隐式链式转换。工程上一般都加 explicit

一个类构造函数如果能“用一个其他类型的值构造本类对象”，并且不是 explicit，就可用于隐式转换。常见是单参数构造。

写起来方便，但是也可能带来误用，所以现代 C++ 常加 explicit。

实例：

~~~cpp
#include <iostream>

class Meter {
public:
    Meter(double v) : value(v) {} // 单参数 且 非 explicit：可隐式转换
    double value;
};

void print_meter(Meter m) {
    std::cout << m.value << " m\n";
}

int main() {
    print_meter(3.5);   // 这里发生隐式转换：double -> Meter 即 3.5 -> Meter
}
~~~

如过改为 `explicit Meter(double v) : value(v) {}`  

`print_meter(3.5);` 报错，必须显示写成  `print_meter(Meter(3.5));`
