sizeof 是编译时的运算符，它属于 c++ 中编译期反射 / 类型查询能力的基础设施，是C++中编译期类型和布局信息的基础工具。服务于代码生成和泛型/元编程。

- `alignof(T)`：编译期得到类型 T 需要对齐的字节数（字节数）
- `decltype(expr)` ： 在编译期决定返回类型
- `std::is_same_v<T,U>`
- `alignas()`：编译时生效的，但不算是运算符，是个对齐说明符。

## 工程中的使用

编译期对类型的对齐约束：
~~~cpp
struct Node {
    int x;
    double y;
};
// sizeof(Node)：Node 类型占的总字节数
// alignof(Node)：Node 需要的对齐字节数
// 编译器布置布局时一定会保证 总大小是对齐的整数倍
static_assert(sizeof(Node) % alignof(Node) == 0, "layout sanity");

#include <type_traits>
static_assert(alignof(double) >= alignof(int), "double 对齐通常不比 int 小");

template<typename T, typename U>
// decltype(a + b) 在编译期“问编译器：这个表达式的类型是什么？”然后把它当作返回类型用，避免开发者猜或硬编码。
// auto 就可以自己推导了，大部分场景不再需要 -> decltype(a + b)
auto add(const T& a, const U& b) -> decltype(a + b) {
    return a + b;
}
~~~
