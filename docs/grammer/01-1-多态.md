## 多态

多态的核心价值：通过**基类指针或引用**操作**派生类对象**，实现统一接口与可扩展实现。

典型场景：新增派生类时，调用方代码不需要改动，只需让新类实现同一组虚函数接口。

~~~cpp
class Animal {  // 抽象基类（接口）
public:
    virtual void speak() = 0;   // 纯虚函数：定义接口，强制派生类实现
    virtual ~Animal() = default; // 关键：多态基类必须有虚析构
};

class Dog : public Animal {
public:
    void speak() override { // override：编译期检查“确实重写了基类虚函数”
        std::cout << "woof" << std::endl;
    }
    ~Dog() override = default;
};

class Cat : public Animal {
public:
    void speak() override {
        std::cout << "meow" << std::endl;
    }
    ~Cat() override = default;  //  不必是virtual，但一定是 override
};

int main() {
    std::unique_ptr<Animal> animal1 = std::make_unique<Dog>(); // 向上转型 + RAII
    std::unique_ptr<Animal> animal2 = std::make_unique<Cat>();
    animal1->speak(); // 动态绑定到 Dog::speak
    animal2->speak(); // 动态绑定到 Cat::speak
}
~~~

语法点速记：

- `virtual`：启用运行时多态（动态绑定）。
- `= 0`：纯虚函数，令类变为抽象类（不能直接实例化）。
- `override`：防止签名写错导致“未重写成功”。
- `virtual ~Base()`：通过基类指针删除派生对象时保证析构完整。
- `std::unique_ptr<Base>`：用 RAII 管理多态对象，减少手动 `delete` 错误。

---

## 静态多态 vs 动态多态

- 静态多态：编译期确定调用目标（函数重载、模板、运算符重载）。
- 动态多态：运行期确定调用目标（虚函数 + 基类指针/引用）。

语法点速记：

- 静态多态通常更易内联、开销更小。
- 动态多态更灵活，但会有间接调用与对象布局开销（实现相关）。

## 父类指针指向子类对象（向上转型）

~~~cpp
class Base {
public:
    virtual void f() { std::cout << "Base::f\n"; }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    void f() override { std::cout << "Derived::f\n"; } // override 表示确实重写了base类的 f()
    void onlyInDerived() { std::cout << "Derived only\n"; }
};

int main() {
    Base * p = new Derived(); // 向上转型：Derived* -> Base*
    p->f();                   // 调虚函数：动态绑定到 Derived::f
    // p->onlyInDerived();    // 错误：Base* 只能直接访问 Base 接口
    delete p;                 // 依赖 Base 的虚析构保证正确析构
}
~~~

语法点速记：

- `Base*` 指向 `Derived` 后，**静态类型**仍是 `Base*`，可见接口由 `Base` 决定。
- 若函数是虚函数，最终调用由对象**动态类型**决定。

## 对象内存布局（概念级）

对于“含虚函数”的对象，常见实现会在对象中放入一个 `vptr`，用于找到该*动态类型对应的* `vtable`。 这属于实现细节，不同编译器/ABI 可能不同。


## vptr & vtable

vtable 是按“类类型”生成的：（多态case下）Base 有自己的 vtable，Derived 也有自己的 vtable。

vptr 是按“对象实例”存在的：每个多态对象里有 vptr，它指向该对象*动态类型对应的* vtable。

父类对象有父类的 vptr ，它指向父类 vtable

子类对象（多态）有子类的 vptr，它指向子类 vtable

当 `Base* p = new Derived;` 时，p 指向的那个对象内部 vptr 通常指向 Derived 的 vtable（因此*虚调用*走到 Derived 实现）

## 析构顺序（对象销毁规则）

对象销毁时，顺序是：

1. 先执行最派生类析构函数体。
2. 再按成员声明的*逆序*析构成员对象。
3. 最后按继承列表*逆序*析构基类子对象。

语法点速记：

- 构造顺序与析构顺序相反。
- 只要你 through base pointer delete，就把基类析构设为 `virtual`。

---

## 一次完整调用链：`Base* p = new Derived(); p->f(); delete p;`

1. `new Derived()`：创建 `Derived` 对象，`p` 向上转型为 `Base*`。  
2. `p->f()`：若 `f` 为虚函数，运行期根据动态类型选择 `Derived::f`。  
3. `delete p`：若基类析构为虚函数，先析构 `Derived`，再析构 `Base`。

语法点速记：

- 动态绑定发生在“通过基类指针/引用调用虚函数”时。
- `delete` 是否安全，取决于基类析构是否 virtual （base类应该是virtual 析构）。

---

## Upcasting（向上转型）

向上转型是把 `Derived*` 隐式转换为 `Base*`，是运行时多态的前提。

```cpp
Derived d;
Base * p = &d; // upcasting
```
向上转型通常是隐式且安全的（语义上“Derived is-a Base”）。


## virtual 总结

- 不需要 virtual 的情况：类*不作为多态基类*，*不会通过base类指针删除派生对象*。可能有这样的使用：`Base* p = new Derived(); delete p;`，Base 的析构必须是 virtual。
- 代价与收益：virtual 析构会让类变成多态类型（通常有 vptr 开销），但这是换取正确析构的必要成本。
- 如果一个类已经有任意虚函数，通常直接把析构也写成 virtual，避免后续误用。
- 只要基类析构是 virtual，子类析构自动也是虚函数（隐式虚）。实战中更推荐子类写 `override` 来表达*我在重写基类虚析构*

总之
- 当类是基类 且 多态删除时，就必须 virtual
- 基类必须 virtual，子类优先写 override
