## 对类的基本认识

类是蓝图，对象是实例。类定义对象的结构和行为，真正占用内存的是对象（实例）。

定义类时，成员初始化常见有三种方式：

- 类内默认成员初始化
- 构造函数初始化列表
- 构造函数函数体内赋值（通常不如初始化列表推荐）

在栈上创建对象：对象有明确生命周期，离开作用域自动析构。

~~~cpp
Student stu;            // 自动存储期对象：离开作用域自动析构
Student * p_stu = &stu; // 指针仅“观察”栈对象，不拥有其生命周期
~~~

在堆上创建对象：`new` 返回动态分配对象的地址，需要手动 `delete` 或使用智能指针托管。

~~~cpp
Student * p_stu = new Student; // 动态分配：对象存活直到 delete
delete p_stu;                  // 释放堆内存并调用析构函数
~~~

语法点速记：

- `new`：在自由存储区（常称堆）创建对象，返回指针。
- `delete`：销毁 `new` 创建的对象；少写会泄漏，重复写是未定义行为。


## 计算一个类对象的大小

类声明本身不占“对象实例内存”，`sizeof(A)` 计算的是 `A` 类型对象大小。

常见规则如下：

1. **空类对象**通常占 1 字节（占位，保证对象地址可区分）。
2. 非静态成员函数（普通成员函数）不计入对象大小；静态成员函数同样不计入对象大小。
3. 非静态数据成员会计入对象大小，并遵循对齐规则。对象总大小通常是其最大对齐需求的整数倍。

    ~~~cpp
    class A {
        int a;  // 4 字节（常见平台）
        char b; // 1 字节，后续可能产生填充字节（padding）
    };
    ~~~

4. 若类含有虚函数，对象中通常会有一个隐藏的*虚表指针* `vptr`（实现相关），其大小通常是 4 或 8 字节。
5. 派生类对象通常包含“基类子对象 + 派生类自身成员”（再加上对齐与实现细节）。
6. *静态成员变量*不存储在对象内，通常位于静态存储区，因此不计入 `sizeof`。

    ~~~cpp
    class A {
        int a;         // 计入对象大小
        static char b; // 不计入对象大小
    };
    ~~~

语法点速记：

- `sizeof(T)`：编译期求类型/对象大小（不执行对象构造逻辑）。
- `static` 成员变量：属于类本身，不属于某个对象实例。
- `virtual`：启用运行时多态（动态绑定），通常引入 `vptr` 开销。

## 多态

多态以*继承*和*虚函数*为基础。通过基类指针/引用调用虚函数时，可在运行时绑定到派生类实现。

- 继承
- 虚函数
- 虚函数表
- RTTI

### 多态相关注意点

- 若类将*被当作基类*并通过基类指针删除对象，应声明**虚析构函数**。
- 若类不是多态基类（不打算通过基类接口多态使用），通常不必引入虚析构函数。
- `virtual` 的意义是*允许派生类重写接口，实现运行时动态绑定*。
- 虚函数代码只存一份，位于代码段。
- 每个多态对象通常含一个 `vptr`，`vptr` 占对象空间。
- `vtable` 通常位于只读数据区（实现相关），不属于对象本体。

语法点速记：

- `virtual ~Base() = default;`：确保 `delete base_ptr` 时调用到派生类析构。
- `override`（配套使用）：让编译器检查“确实重写了基类虚函数”。

## 析构函数

析构函数没有参数、不能重载，因此一个类只能有一个析构函数。

## 析构函数的执行时机

析构函数在对象销毁时调用，时机取决于对象生命周期：

- 全局对象：程序结束时析构。
- 局部对象（自动存储期）：离开作用域时析构。
- `new` 创建的对象：`delete` 时析构；若未 `delete` 则资源泄漏。

语法点速记：

- 析构函数名固定为 `~ClassName()`，无参数、无返回值。
- 推荐 RAII：把资源绑到对象生命周期，减少手动 `delete`。

## `struct` 与 `class`

两者语法能力基本一致（都可有成员函数、模板等），主要差异是默认访问权限：

- `struct` 默认 `public`
- `class` 默认 `private`

## 类本身是一个作用域

在类作用域之外：

- 普通成员通常通过对象访问。
- 静态成员可通过对象或*类名访问*（推荐类名访问）。
- 类型别名（`typedef` / `using`）受类作用域限制，需带作用域限定符访问（如 `A::value_type`）。

## 常对象与 `const` 成员函数

`const` 可修饰对象。`const` 对象（或指向 `const` 对象的指针）只能调用 `const` 成员函数。

`const` 成员函数*承诺不修改*对象的逻辑状态（除 `mutable` 成员外）。

~~~cpp
class Student {
public:
    Student(char * name, int age, float score);
    void show();                                 // 非 const 成员函数：可能修改对象
    char * getname() const;                      // const 成员函数：承诺不改对象逻辑状态
    int getage() const;
    float getscore() const;

private:
    char * m_name;
    int m_age;
    float m_score;
};

// 初始化列表：成员在进入函数体前完成初始化
Student::Student(char * name, int age, float score) : m_name(name), m_age(age), m_score(score) {}

void Student::show() {
    cout << m_name << " age is: " << m_age << ", score is: " << m_score << endl;
}

char * Student::getname() const {
    // 函数尾部 const 约束的是 this 指针：this 视为指向 const 对象
    return m_name;
}

int Student::getage() const {
    return m_age;
}

float Student::getscore() const {
    return m_score;
}

int main() {
    const Student stu("Xiaoming", 15, 90.6f); // 常对象
    // stu.show(); // error 因为 show() 不是 const 成员函数，所以不能被 const 对象或指向 const 对象的指针调用。
    cout << stu.getname() << " age is: " << stu.getage() << ", score: " << stu.getscore() << endl;

    const Student * pstu = new Student("Lili", 16, 80.5f); // 指向常对象的指针
    // pstu->show(); // error
    cout << pstu->getname() << " age is: " << pstu->getage() << ", score: " << pstu->getscore() << endl;
    delete pstu;
    return 0;
}
~~~

语法点速记：

- `const Student stu;`：对象不可被非常量成员函数修改。
- `const Student * p`：`p` 可改指向，但不能通过 `p` 修改对象。const 的是 Student
- `Student * const p`：`p` 不可改指向，但可通过 `p` 修改对象。const 的是指针 p

## `mutable`

`mutable` 允许在 `const` 成员函数中修改特定成员，常用于缓存、统计计数等“不影响对象逻辑状态”的数据。

~~~cpp
class MyClass {
public:
    int getSomething() const {
        callCount++; // mutable 成员可在 const 函数中修改
        return something;
    }

    int getCallCount() const {
        return callCount;
    }

private:
    int something = 0;
    mutable int callCount = 0; // 计数器不属于“逻辑常量性”
};
~~~

语法点速记：

- `mutable`：只是在语法层面允许你在 const 成员函数里修改某个成员（即“位常量性”放宽）。但从设计语义上，这个修改*不应该让外部观察*到“对象核心状态变了”。常见场景：缓存、命中次数、延迟计算标记。这些不改变对象“本质值”。

## 嵌套类

嵌套类（nested class）是定义在类内部的类。常用于增强封装、组织代码和减少命名污染。

## 避免创建匿名堆对象

`(new Student("Michael", 90))->Show();` 这类写法会造成内存泄漏（分配后无处释放）。

推荐做法：

1. 优先在栈上创建对象。
2. 需要动态分配时使用智能指针（RAII）：

    ~~~cpp
    std::unique_ptr<Student> student = std::make_unique<Student>("Michael", 90);
    student->Show();
    ~~~

语法点速记：

- `std::unique_ptr<T>`：独占所有权，不可拷贝、可移动。
- `std::make_unique<T>(...)`：异常安全地创建对象并交给智能指针管理。

## 类的元数据

编译器在编译期间会维护类的类型信息与布局信息（可理解为“类的元数据”），例如：

- 类名与继承关系
- 成员变量与成员函数信息
- 构造/析构相关信息
- 访问控制信息
- 模板参数信息
- 内存布局（偏移、对齐、大小）
- 多态类相关的虚表信息（实现相关）

运行时并不存在一个统一、可直接遍历的“类元数据对象”。但对于含虚函数的类，目标文件中通常会保留支持动态分发所需的虚表相关信息（实现相关）。
